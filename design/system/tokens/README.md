## This directory is the single source of truth for all design tokens.

No design values (colors, spacing, radii, typography, shadows, motion) may be hardcoded in frontend source code. All values must reference `--tkn-*` CSS custom properties generated by Style Dictionary from these JSON files.

## Token Files

| File | Contents | Key tokens |
|------|----------|------------|
| `color.json` | Core palette · semantic colors · subject palette · platform overrides | `--tkn-color-*` · `--tkn-subject-*` |
| `typography.json` | Font families · full type scale (14 levels) | `--tkn-font-*` · `--tkn-type-*` |
| `spacing.json` | 4pt/8pt grid (2px–96px) | `--tkn-spacing-*` |
| `radius.json` | Border radius scale (5px → pill → circle) | `--tkn-radius-*` |
| `shadow.json` | Card shadow · nav glass · focus ring | `--tkn-shadow-*` |
| `motion.json` | Apple HIG easing curves · 150/250/400ms durations | `--tkn-motion-*` |

## Naming Convention

All generated CSS custom properties follow the pattern:

```
--tkn-<group>-<name>-<variant>
```

Examples:
- `--tkn-color-primary-DEFAULT` → Apple Blue `#0071e3`
- `--tkn-subject-math` → Subject carmine red `#C41E3A`
- `--tkn-spacing-md` → 16px
- `--tkn-radius-pill` → 980px
- `--tkn-motion-ease-apple-standard` → `cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- `--tkn-motion-duration-fast` → 150ms

## Platform Overrides

WeChat miniprogram requires saturation and shadow corrections due to gamma rendering differences. These are declared in `color.json._meta.platform_overrides` and applied by the Style Dictionary wxss transform. The JSON source remains platform-neutral — differences appear only in `tokens.wxss`.

## A11y Contract

Every semantic color in `color.json` carries `contrast_on_light` and `contrast_on_dark` fields (WCAG formula, calculated by `node -e` script at token authoring time). Colors flagged with `_warning` indicate which surface pairings must be avoided in component code.
